# Organizations (Orgs) Implementation Design

## Goal

Group multiple admins under a shared org with a shared Google Calendar, role-based permissions, an invite flow, and org-aware booking with priority-based admin selection.

## Architecture

### New modules
- `app/models/org.py` — Org model
- `app/models/org_member.py` — OrgMember join table (customer ↔ org, role, priority)
- `app/models/org_invite.py` — pending org invites
- `app/auth.py` — `require_auth` decorator + `generate_auth_url` helper
- `app/org/__init__.py`, `app/org/routes.py` — org blueprint
- `app/org/email.py` — invite email helper
- `app/org/selection.py` — admin selection logic (freebusy + priority)

### Modified modules
- `app/models/customer.py` — add `api_token` field
- `app/main/routes.py` — remove `GET /`; update `GET /callback` to return Bearer token
- `app/booking/routes.py` — `POST /book` uses `org_id` + optional `customer_id`; `GET /confirm-booking/<token>` creates event on both admin and org calendar
- `app/__init__.py` — register org blueprint, import new models
- `migrations/versions/002_add_orgs.py` — Alembic migration for all schema changes

### Tech stack
- Existing: Flask, SQLAlchemy, Flask-Mail, google-api-python-client, dnspython
- No new dependencies

## Authentication

### How it works
- `Customer.api_token` is generated by `secrets.token_urlsafe(32)` after a successful OAuth flow.
- Protected routes require `Authorization: Bearer <token>` header.
- `require_auth` decorator in `app/auth.py` looks up the customer by token and attaches it to `flask.g.current_customer`. Returns `401` with a fresh Google OAuth URL if missing or invalid.
- `GET /` is **removed**.

### `GET /callback` (modified)
After OAuth completes and credentials are saved, generate (or regenerate) `api_token` and return:
```json
{"token": "<api_token>"}
```

### `require_auth` decorator
```python
def require_auth(f):
    @wraps(f)
    def decorated(*args, **kwargs):
        auth_header = request.headers.get('Authorization', '')
        token = auth_header.removeprefix('Bearer ').strip()
        customer = Customer.query.filter_by(api_token=token).first() if token else None
        if not customer:
            auth_url = generate_auth_url()
            return jsonify({'error': 'Unauthorized', 'auth_url': auth_url}), 401
        g.current_customer = customer
        return f(*args, **kwargs)
    return decorated
```

`generate_auth_url()` builds a fresh Google OAuth URL using `_build_flow()`. Move `_build_flow` from `app/main/routes.py` to `app/google_calendar.py` so both `app/auth.py` and `app/main/routes.py` can import it without circular dependencies.

### Protected routes
- `POST /org/register`
- `POST /org/invite`
- `PUT /org/members/<customer_id>`
- `POST /book`

Public routes (no auth required):
- `GET /callback` — this IS the auth endpoint
- `GET /org/join/<token>` — guest/invitee clicking email link
- `GET /org/join-callback` — OAuth redirect target for invite flow
- `GET /confirm-booking/<token>` — guest clicking booking confirmation link
- `GET /get-busy-hours` — read-only, unchanged

## Org Registration

### `POST /org/register` (protected)
Body: `{ "org_name": "..." }`

Steps:
1. Verify `g.current_customer` has no existing `OrgMember` record (cannot register a second org).
2. Call Google Calendar API `calendars().insert(body={"summary": org_name})` using the customer's credentials to create the shared calendar.
3. Call `acl().insert()` to grant the customer write access to the new calendar.
4. Create `Org(name=org_name, google_calendar_id=<new_calendar_id>)`.
5. Create `OrgMember(org=org, customer=g.current_customer, role='owner', priority=1)`.
6. Return `{"org_id": org.id, "calendar_id": org.google_calendar_id}`.

## Invite Flow

### `POST /org/invite` (protected, manager+)
Body: `{ "invitee_email": "...", "role": "employee", "priority": 1 }`

Steps:
1. Load `OrgMember` for `g.current_customer`. Must be `owner` or `manager`. `403` otherwise.
2. Managers can only assign role `employee`. Owners can assign any role including `owner` (promote) and can demote another owner to `manager` or `employee`.
3. Check no `OrgMember` exists for `invitee_email` (i.e., no Customer with that email already has an OrgMember). Return `400 "User already belongs to an org"` if so.
4. Check no existing non-expired `OrgInvite` for `(org_id, invitee_email)`. Return `400 "Invite already pending"` if so.
5. Create `OrgInvite(org_id, invited_email, role, priority, token=secrets.token_urlsafe(32), expires_at=utcnow+24h)`.
6. Send invite email with link `<BOOKING_CONFIRM_BASE_URL>/org/join/<token>`.
7. Return `200 {"message": "Invite sent to <email>"}`.

### `GET /org/join/<token>` (public)
1. Look up `OrgInvite` by token. `404` if not found.
2. Check not expired. `410` if expired.
3. Store token in `session['org_invite_token']`.
4. Build OAuth flow with `redirect_uri=/org/join-callback`.
5. Redirect to Google OAuth URL.

### `GET /org/join-callback` (public — OAuth redirect target)
1. Complete OAuth, fetch credentials and email from Google.
2. Retrieve `session['org_invite_token']`; look up `OrgInvite`. `400` if missing or expired.
3. Verify `user_email == org_invite.invited_email`. `400 "Email mismatch"` if not.
4. Look up existing `Customer` by email:
   - If exists and already has `OrgMember` for the **same org**: update credentials, clear invite, return token. (Re-link case.)
   - If exists and has `OrgMember` for a **different org**: return `400 "Already belong to another org"`.
   - Otherwise: create or update `Customer` with credentials + new `api_token`.
5. Create `OrgMember(org_id, customer, role=invite.role, priority=invite.priority)`.
6. Grant write ACL on `org.google_calendar_id` for the new member.
7. Delete `OrgInvite`.
8. Clear `session['org_invite_token']`.
9. Return `{"token": customer.api_token}`.

## Member Management

### `PUT /org/members/<customer_id>` (protected)
Body: `{ "role": "...", "priority": 1 }` — all fields optional.

Permission matrix:

| Actor role | Target | Can update role? | Can update priority? |
|---|---|---|---|
| owner | any other member | yes (any role) | yes |
| manager | employee | yes (`employee` only) | yes |
| manager | manager or owner | no — 403 | no |
| employee | themselves | no | no |

Note: no customer delete endpoint in this release. Soft-delete will be added later.

Steps:
1. Load actor's `OrgMember`. Must belong to an org. Load target's `OrgMember`. Must be in the same org.
2. Apply matrix. Return `403` for violations.
3. Apply updates. `db.session.commit()`.
4. Return `200 {"message": "Updated"}`.

## Booking Changes

### `POST /book` (protected)
Body replaces `store_email` with `org_id`; adds optional `customer_id`.

```json
{
  "org_id": 1,
  "customer_id": 42,        // optional — pick a specific admin
  "guest_email": "...",
  "guest_name": "...",
  "date": "2024-08-01",
  "start_time": "11:00",
  "end_time": "12:30"
}
```

Steps:
1. Validate fields, parse datetimes, check alignment and duration (unchanged).
2. Look up `Org` by `org_id`. `400` if not found.
3. MX-validate `guest_email` (unchanged).
4. Load all `OrgMember`s for the org. `400` if org has no members.
5. **Admin selection** (see below). Returns chosen `Customer` or `409` if none available.
6. Create `PendingBooking` with `admin_customer_id` (new FK field) instead of `store_email`.
7. Send confirmation email to guest.
8. Return `201 {"message": "Confirmation email sent to <email>"}`.

### Admin selection logic (`app/org/selection.py`)

```python
def select_admin(members, start_dt, end_dt, preferred_customer_id=None):
    # 1. Batch freebusy query for all members' calendars
    # 2. Filter to free members
    # 3. If preferred_customer_id given: check they're free → return them or 409
    # 4. Sort free members by priority descending
    # 5. Pick randomly among those sharing the highest priority
    # Returns Customer or None
```

The freebusy check queries each member's personal calendar (`calendarId='primary'`) using their stored credentials.

### `GET /confirm-booking/<token>` (public, modified)
After creating the event on the admin's personal calendar (existing logic), also call:
```python
service_org = build('calendar', 'v3', credentials=admin_credentials)
service_org.events().insert(
    calendarId=org.google_calendar_id,
    body=event_body,
    sendUpdates='none',   # org calendar is internal, no extra notifications
).execute()
```

`PendingBooking` gains two new fields: `admin_customer_id` (FK → Customer) and `org_id` (FK → Org) so `confirm_booking` can load them.

## Error Reference

| Code | Meaning |
|---|---|
| 400 | Validation failure, org not found, email mismatch, already in org |
| 401 | Missing or invalid Bearer token — includes `auth_url` |
| 403 | Permission denied (role hierarchy violation) |
| 404 | Token not found |
| 409 | No admin available / slot taken |
| 410 | Invite or booking confirmation link expired |
